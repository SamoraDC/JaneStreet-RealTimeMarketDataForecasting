## What Each Part Does

- `multi-models/run_strong_oof_experiment.py`: CLI entrypoint for the strong OOF
  modular experiment.
- `multi-models/multimodels/strong_oof.py`: implements gateway RLS features,
  residual corrections, residual-tail masks, fixed blends, scoring, reports, and
  anti-leakage audit payloads.
- `scripts/run_bayesian_gateway_meta_simulation.py`: builds gateway-style expert
  predictions and feature sets used by strong OOF.
- `scripts/run_dynamic_gateway_rls_validation.py`: provides the causal dynamic
  RLS update used by the gateway candidates.
- `scripts/run_gateway_rls_strategy_selection.py`: provides preserved RLS
  strategies and risk-modulated shrinkage candidates.
- `src/janestreet/bayesian_meta.py`, `blending.py`, `metrics.py`: shared scoring
  and blending utilities used by the scripts.
- `multi-models/tests/test_strong_oof.py`: focused tests for the strong OOF
  machinery.

## Reproduction Command

The preserved historical result is generated by the strong OOF experiment over
historical `max_date_id=1398` OOF predictions, with residual-tail modes enabled:

```bash
uv run python multi-models/run_strong_oof_experiment.py \
  --experiment-name strong_oof_hist_max1398_gateway_residual_tail_modes_v1 \
  --tabm-prediction-dir reports/experiments/competitive_tabm_official_stage3_hist_max1398_5fold_valid60_lags_online_lr1e4_4m_train700_seed37_aux8_preds/validation_predictions \
  --tree-prediction-dir reports/experiments/tree_engine_ensemble_hist_max1398_xgb_lgb_sample10_seed_ensemble_preds/validation_predictions \
  --output-dir multi-models/reports/strong_oof_hist_max1398_gateway_residual_tail_modes_v1 \
  --include-gateway-risk-shrink \
  --residual-base-candidates gateway_risk_conservative_rls_abs_pred_s100_prediction,gateway_risk_aggressive_rls_abs_pred_s25_prediction \
  --residual-tail-quantiles 0.90,0.95,0.99 \
  --residual-tail-modes weight,abs_base,disagreement,weight_or_abs,weight_and_abs,weight_or_disagreement
```

The best preserved candidate from that report is:

```text
gateway_risk_conservative_rls_abs_pred_s100_prediction_residual_weight_and_abs_q0p95_residual_tail
```

## Notes

This is an OOF/reference implementation. It is not yet a Kaggle runtime
submission package. Making it submissible requires exporting the residual
coefficients and tail thresholds into the runtime and applying the same causal
logic inside `submission.py`.
